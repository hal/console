데이터 소스 서브시스템을 사용하면 데이터 소스를 생성 및 구성하고 JDBC 데이터베이스 드라이버를 관리할 수 있습니다.

데이터 소스

두 가지 일반적인 유형의 리소스를 데이터 소스와 XA 데이터 소스라고 합니다.

  • 데이터 소스 는 트랜잭션을 사용하지 않는 애플리케이션 또는 단일 데이터베이스와의 트랜잭션을 사용하는 애플리케이션에 사용됩니다.
  • XA 데이터 소스 는 트랜잭션이 여러 데이터베이스에 분산되어 있는 애플리케이션에서 사용됩니다. XA 데이터 소스는 추가 오버헤드를 발생시킵니다.

JDBC 드라이버

애플리케이션이 데이터 소스에 연결되기 전에 데이터 소스 공급업체의 JDBC 드라이버를 설치해야 합니다. JDBC 드라이버를 설치하는 두 가지 방법 중에서 선택할 수 있습니다.

모듈

JDBC 드라이버를 모듈로 설치하려면 다음 아래에 파일 경로 구조를 생성해야 합니다. ${build.installDir}/modules, JDBC 드라이버 JAR을 main/ 하위 디렉터리를 만들고 module.xml 파일을 만듭니다.

JDBC 드라이버를 모듈로 사용할 수 있게 되면 이 섹션을 사용하여 드라이버 구성을 추가하고 제거할 수 있습니다.

배포

다른 배포와 마찬가지로 JDBC 드라이버를 배포할 수 있습니다. 즉, 관리형 도메인을 사용하는 경우 서버 그룹의 여러 서버에 배포할 수 있습니다. 모든 JDBC 타입4 규격 드라이버는 이름과 버전별로 자동으로 인식되어 시스템에 설치됩니다.

도메인 모드에서 애플리케이션으로 배포된 드라이버는 선택한 프로필과 일치하는 서버가 실행 중인 경우에만 이 섹션에 표시됩니다.
