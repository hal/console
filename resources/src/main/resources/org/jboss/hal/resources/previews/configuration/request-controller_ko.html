${build.shortName}은(는) 일시중단되거나 정상적으로 종료될 수 있습니다. 이를 통해 새로운 요청을 수락하지 않고 활성 요청을 정상적으로 완료할 수 있습니다. 시간 초과 값은 일시 중지 또는 종료 작업이 활성 요청이 완료될 때까지 기다리는 시간을 지정합니다. 서버가 일시 중지된 동안에도 관리 요청은 계속 처리됩니다.

단계적 종료는 서버 전체 수준에서 조정되며 주로 요청이 서버에 들어오는 진입점에 중점을 둡니다. 다음 서브시스템은 단계적 종료를 지원합니다:

undertow
undertow 서브시스템은 모든 요청이 완료될 때까지 기다립니다.
모드클러스터
modcluster 서브시스템은 서버가 일시 중지 중임을 로드 밸런서에 알립니다. PRE_SUSPEND 단계.
EJB
ejb3 서브시스템은 모든 원격 EJB 요청 및 MDB 메시지 전달이 완료될 때까지 기다립니다. MDB로의 전달이 중단되었습니다. PRE_SUSPEND 단계. EJB 타이머는 일시 중지되며, 누락된 타이머는 서버가 재개되면 활성화됩니다.
EE 동시성
서버는 모든 활성 작업이 완료될 때까지 기다립니다. 대기 중인 모든 작업을 건너뜁니다. 현재 EE Concurrency에는 지속성이 없기 때문에 건너뛰고 대기 중인 작업은 손실됩니다.
서버가 일시 중지된 동안 예약된 작업은 예약된 시간에 계속 실행되지만 오류가 발생합니다. java.lang.IllegalStateException. 서버가 다시 시작되면 예약된 작업은 계속해서 정상적으로 실행되며 대부분의 경우 작업을 다시 예약할 필요가 없습니다.
배치
서버는 제한 시간 내에 실행 중인 모든 작업을 중지하고 예약된 모든 작업을 연기합니다.
현재 단계적 종료는 인바운드 원격 분산 트랜잭션이나 새로운 인바운드 JMS 메시지를 거부하지 않습니다. 진행 중인 활동에 의해 예약된 EE 배치 작업 및 EE 동시성 작업은 현재 진행이 허용됩니다. 그러나 현재 시간 제한 창을 통과하는 제출된 EE 동시성 작업은 실행 시 오류가 발생합니다.

요청은 요청 컨트롤러 서브시스템에 의해 추적됩니다. 이 서브시스템이 없으면 일시 중지 및 재개 기능이 제한되며 서버는 일시 중지 또는 종료되기 전에 요청이 완료될 때까지 기다리지 않습니다. 그러나 이 기능이 필요하지 않은 경우 약간의 성능 향상을 위해 요청 컨트롤러 서브시스템을 제거할 수 있습니다.
