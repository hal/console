보안 도메인(security domain)은 인증, 권한 부여, 보안 매핑 및 감사에 대한 구성으로 구성됩니다. JAAS(Java Authentication and Authorization Service) 선언적 보안을 구현합니다.

인증(Authentication)이란 사용자의 신원을 확인하는 것을 의미합니다. 보안 용어에서는 이 사용자를 주체라고 합니다. 인증과 권한 부여는 다르지만 포함된 인증 모듈 중 상당수는 권한 부여도 처리합니다.

승인(Authorization)은 인증된 사용자가 시스템이나 작업의 특정 리소스에 액세스할 수 있는 권한이 있는지 서버가 결정하는 프로세스입니다.

보안 매핑(Security mapping)은 정보를 애플리케이션에 전달하기 전에 주체, 역할 또는 속성에서 정보를 추가, 수정 또는 삭제하는 기능을 의미합니다.

감사 관리자(The auditing manager)를 사용하면 보안 이벤트가 보고되는 방식을 제어하도록 프로바이더 모듈을 구성할 수 있습니다. 보안 도메인을 사용하는 경우 애플리케이션 자체에서 특정 보안 구성을 모두 제거할 수 있습니다. 이를 통해 보안 매개변수를 중앙에서 변경할 수 있습니다. 이러한 유형의 구성 구조를 활용하는 일반적인 시나리오 중 하나는 테스트 환경과 프로덕션 환경 간에 애플리케이션을 이동하는 프로세스입니다.
