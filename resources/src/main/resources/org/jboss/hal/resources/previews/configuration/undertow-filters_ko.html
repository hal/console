필터를 사용하면 요청의 일부 측면을 수정할 수 있으며 조건자를 사용하여 필터 실행 시기를 제어할 수 있습니다. 필터의 일반적인 사용 사례에는 헤더 설정 또는 GZIP 압축 수행이 포함됩니다.

다음 유형의 필터를 정의할 수 있습니다:

  1. 커스텀 필터 (Custom Filter)
  2. 에러 페이지 (Error Page)
  3. 표현식 필터 (Expression Filter)
  4. GZip
  5. Mod Cluster
  6. 요청 제한 (Request Limit)
  7. 응답 헤더 (Response Header)
  8. 재작성 (Rewrite)

참고
필터는 이전 버전의 ${build.shortName}에서 사용된 전역 밸브와 기능적으로 동일합니다.
