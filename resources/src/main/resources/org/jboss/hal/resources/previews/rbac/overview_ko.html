RBAC(역할 기반 액세스 제어)는 관리 사용자에 대한 권한 집합을 지정하는 메커니즘입니다. 이를 통해 여러 사용자가 각각 무제한 액세스를 요구하지 않고도 서버 관리 책임을 공유할 수 있습니다. 관리 사용자에게 "업무 분리"를 제공함으로써 불필요한 권한을 부여하지 않고도 개인이나 그룹 간에 책임을 쉽게 분산시킬 수 있습니다. 이를 통해 서버와 데이터의 보안을 최대한 보장하는 동시에 구성, 배포, 관리에 유연성을 제공합니다.

역할 기반 액세스 제어는 역할 권한과 제약 조건의 조합을 통해 작동합니다. 각각 서로 다른 고정 권한을 갖는 사전 정의된 7개의 역할이 제공됩니다. 사전 정의된 역할은 모니터, 운영자, 유지 관리자, 배포자, 감사자, 관리자 및 수퍼유저입니다(권한에 대한 자세한 내용을 보려면 역할을 선택하십시오). 각 관리 사용자에게는 서버 관리 시 사용자에게 허용되는 작업을 지정하는 하나 이상의 역할이 할당됩니다.

중요: 프로바이더를 rbac로 변경하기 전에 구성에 RBAC 역할 중 관리자 또는 수퍼유저 역할에 매핑된 사용자가 있는지 확인하세요. 그렇지 않으면 설치를 종료하고 XML 구성을 편집하지 않는 한 설치를 관리할 수 없습니다.
