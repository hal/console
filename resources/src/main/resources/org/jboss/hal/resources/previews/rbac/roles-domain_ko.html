표준 역할

사전 정의된 사용자 역할은 7가지가 있습니다.

  • 모니터(monitor)
  • 운영자(operator)
  • 유지관리자(maintainer)
  • 배포자(deployer)
  • 감사자(auditor)
  • 관리자(administrator)
  • 수퍼유저(superuser)

이러한 각 역할은 서로 다른 권한 집합을 가지며 특정 사용 사례에 맞게 설계되었습니다. 모니터, 운영자, 유지 관리자, 관리자 및 수퍼유저 역할은 각각 이전 역할을 기반으로 하며 이전 역할보다 더 많은 추가 권한을 갖습니다. 감사자 및 배포자 역할은 각각 모니터 및 유지 관리자 역할과 유사하지만 몇 가지 추가 특수 권한 및 제한 사항이 있습니다.

범위가 지정된 역할

범위 지정 역할은 하나 이상의 지정된 서버 그룹이나 호스트에 대해서만 표준 역할 중 하나의 권한을 부여하는 사용자 정의 역할입니다. 범위가 지정된 역할을 통해 관리 사용자는 필요한 서버 그룹 또는 호스트로만 제한되는 권한을 부여받을 수 있습니다.

이는 다섯 가지 특성으로 정의됩니다:

  1. 고유한 이름 정의
  2. 표준 역할을 기반 정의
  3. 서버 그룹이나 호스트별 적용
  4. 서버 그룹이나 호스트별 제한
  5. 모든 사용자가 자동으로 포함되는 경우. 기본값은 false

범위가 지정된 역할은 표준 역할과 동일한 방식으로 사용자 및 그룹에 할당될 수 있습니다.

범위가 지정된 역할을 생성해도 새 권한을 정의할 수는 없습니다. 범위가 지정된 역할은 제한된 범위에서 기존 역할의 권한을 적용하는 데에만 사용할 수 있습니다. 예를 들어 단일 서버 그룹으로 제한되는 배포자 역할을 기반으로 범위가 지정된 역할을 생성할 수 있습니다.

역할을 제한할 수 있는 범위는 호스트와 서버 그룹 두 가지뿐입니다.

호스트 범위 역할
호스트 범위 역할은 해당 역할의 권한을 하나 이상의 호스트로 제한합니다. 이는 관련 /host=*/ 리소스 트리에 대한 액세스가 제공되지만 다른 호스트에 특정한 리소스는 숨겨짐을 의미합니다.
서버 그룹 범위 역할
서버 그룹 범위의 역할은 해당 역할의 권한을 하나 이상의 서버 그룹으로 제한합니다. 또한 역할 권한은 지정된 서버 그룹과 연결된 프로필, 소켓 바인딩 그룹, 서버 구성 및 서버 리소스에도 적용됩니다. 서버 그룹과 논리적으로 관련되지 않은 하위 리소스는 사용자에게 표시되지 않습니다.
