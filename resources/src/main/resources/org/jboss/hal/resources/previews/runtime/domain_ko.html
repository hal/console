도메인은 하나의 도메인 컨트롤러, 하나 이상의 호스트 컨트롤러, 호스트당 0개 이상의 서버 그룹으로 구성됩니다.

도메인 컨트롤러는 도메인을 제어하는 ​​중심점입니다. 이는 각 서버가 도메인의 관리 정책에 따라 구성되었는지 확인합니다. 도메인 컨트롤러는 호스트 컨트롤러이기도 합니다.

호스트 컨트롤러는 domain.sh 또는 domain.bat 스크립트가 실행되는 물리적 또는 가상 호스트입니다. 호스트 컨트롤러는 도메인 관리 작업을 도메인 컨트롤러에 위임하도록 구성됩니다. 각 호스트의 호스트 컨트롤러는 도메인 컨트롤러와 상호 작용하여 호스트에서 실행 중인 애플리케이션 서버 인스턴스의 수명 주기를 제어하고 도메인 컨트롤러가 이를 관리하도록 지원합니다. 각 호스트에는 여러 서버 그룹이 포함될 수 있습니다.

서버 그룹은 ${build.shortName}이 설치되어 하나로 관리 및 구성되는 서버 인스턴스 집합입니다. 도메인 컨트롤러는 서버 그룹의 구성과 서버 그룹에 배포된 응용 프로그램을 관리합니다. 결과적으로 서버 그룹의 각 서버는 동일한 구성과 배포를 공유합니다.
