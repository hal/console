호스트 컨트롤러는 domain.sh 또는 domain.bat 스크립트가 호스트에서 실행될 때 시작됩니다.

호스트 컨트롤러의 주요 책임은 서버 관리입니다. 도메인 관리 작업을 위임하고 해당 호스트에서 실행되는 개별 애플리케이션 서버 프로세스를 시작하고 중지하는 일을 담당합니다.

도메인 컨트롤러와 상호작용하여 서버와 도메인 컨트롤러 간의 통신을 관리하는 데 도움을 줍니다. 도메인의 여러 호스트 컨트롤러는 단일 도메인 컨트롤러와만 상호 작용할 수 있습니다. 따라서 단일 도메인 모드에서 실행되는 모든 호스트 컨트롤러와 서버 인스턴스는 단일 도메인 컨트롤러를 가지며 동일한 도메인에 속해야 합니다.

기본적으로 각 호스트 컨트롤러는 domain/configuration/host.xml 파일은 호스트 파일 시스템의 압축이 풀린 ${build.shortName} 설치 파일에 있습니다. 그만큼 host.xml 파일에는 특정 호스트와 관련된 다음 구성 정보가 포함되어 있습니다.

  • 이 설치에서 실행하려는 ${build.shortName} 인스턴스의 이름입니다.
  • 다음 구성 중 하나:
    • 호스트 컨트롤러가 도메인 컨트롤러에 알리고 호스트 컨트롤러 자체를 등록하고 도메인 설정에 액세스하는 방법
    • 원격 도메인 컨트롤러 검색 및 알림 방법
    • 호스트 컨트롤러가 도메인 컨트롤러로 작동하는지
  • 로컬 물리적 설치와 관련된 구성입니다. 예를 들어, domain.xml에 선언된 명명된 인터페이스 정의는 host.xml의 실제 시스템별 IP 주소에 매핑될 수 있습니다. 그리고 domain.xml의 추상 경로 이름은 host.xml의 실제 파일 시스템 경로에 매핑될 수 있습니다.
